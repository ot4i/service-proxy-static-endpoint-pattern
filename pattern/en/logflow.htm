Log subflow for the Service Proxy: static endpoint pattern

When logging is enabled, the Log subflow logs requests and replies, according to the user-defined properties: RequestLoggingOn and ResponseLoggingOn.

Log messages are written as persistent WebSphere MQ messages to a WebSphere MQ queue. These log messages are processed separately in accordance with the standard practices within your organization. The message log contains the original message and the following information in an MQRFH2 header:

  • Time stamp
  • Broker name
  • Message flow name
  • Service details
  • Status (1 = request, 2 = reply, -2 = fault)
