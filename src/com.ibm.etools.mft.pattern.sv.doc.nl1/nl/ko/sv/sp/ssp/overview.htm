서비스 프록시: 정적 엔드 포인트 패턴

이 패턴을 사용하면 대상 서비스 제공자에 직접 바인딩된 가상 서비스를 통해 라우팅하여 웹 서비스 요청자와 웹 서비스 제공자 간에 디커플링을 제공할 수 있습니다.

다음과 같은 경우 이 패턴을 사용하십시오.

  • 서비스가 위치하는 엔드 포인트의 주소를 변경하거나 서비스의 액세스를 제어하기 위해 해당 주소를 숨겨야 하기 때문에 클라이언트가 서비스에 직접 액세스하지 않게 하려는 경우
  • HTTP 또는 HTTPS를 통한 서비스 액세스를 지원하려는 경우
  • 서비스 제공자에 대한 모든 요청의 로그 기록을 제공하거나 서비스 제공자를 변경하지 않고 추가 오류 핸들링 기능을 제공하려는 경우

솔루션

솔루션은 제공자 웹 서비스의 프록시를 제공하고 로그 기록 및 오류 핸들링 기능도 포함할 수 있는 메시지 플로우를 구현하는 것입니다.
