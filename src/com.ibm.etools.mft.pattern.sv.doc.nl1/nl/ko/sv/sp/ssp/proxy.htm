서비스 프록시: 정적 엔드포인트 패턴의 프록시 플로우 세부사항입니다.

SimpleServiceProxy 플로우는 다음과 같은 방식으로 모든 서비스 요청을 처리합니다.

  1. 메시지 플로우에서 요청을 수신할 때 패턴 인스턴스 구성이 유효성 검증을 필요로 하는 경우 요청을 유효성 검증합니다.
  2. 메시지 플로우는 패턴 매개변수에 제공되거나 브로커 아카이브(BAR) 파일에 구성된 제공자 URL을 사용하여 서비스 요청을 대상 서비스 제공자로 전달하는 데 필요한 매개변수를 설정합니다.
  3. 그런 다음 SOAP 요청이 서비스 요청자로 송신됩니다. 성공적인 응답이 수신되면 이 응답이 시작 서비스 요청자로 직접 리턴됩니다. 결함 메시지가 수신되면 플로우 상태가 결함의 발생을 알리도록 설정되고 결함 메시지가 서비스 요청자로 리턴됩니다.
  4. 요청 및 응답의 로그 기록을 포함하거나 제외하도록 패턴을 구성할 수 있습니다.
  5. 로그 기록이 포함되어 있지 않으면 응답이 리턴될 때 메시지 플로우가 성공적으로 종료됩니다.
  6. 메시지 플로우는 오류 핸들링을 위한 서브플로우도 포함하고 있습니다.
  7. 패턴 인스턴스에 대해 이 옵션을 구성한 경우 오류 메시지가 오류 큐에 기록됩니다.
