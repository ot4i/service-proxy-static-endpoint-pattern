서비스 프록시: 정적 엔드포인트 패턴을 생성한 후 완료할 태스크

해당 패턴을 생성한 후 완료해야 하는 태스크입니다.

  1. 브로커 아카이브(BAR) 파일을 작성하십시오. 메시지 플로우가 브로커 아카이브 파일에 추가되어야 합니다.
  2. 브로커 아카이브 파일을 구성하십시오. 서비스 프록시: 정적 엔드포인트 패턴 인스턴스를 위한 메시지 플로우를 배치하기 전에 브로커 아카이브 파일에서 메시지 플로우를 추가 구성할 필요는 없지만, 몇 가지 기본 옵션을 검토하고 사용자 환경에 맞게 구성해야 합니다.

    브로커 아카이브 편집기의 관리 페이지에서:

    • 선택사항: RequestLoggingOn 사용자 정의 특성을 사용하여 메시지 플로우에 대한 로그 기록 요청의 사용 여부를 선택하십시오.
    • 선택사항: ResponseLoggingOn 사용자 정의 특성을 사용하여 메시지 플로우에 대한 로그 기록 응답의 사용 여부를 선택하십시오.
    • 선택사항: ErrorLoggingOn 사용자 정의 특성을 사용하여 오류 메시지의 생성 여부를 선택하십시오.
    • 선택사항: 브로커 아카이브에서 구성 가능한 특성을 변경할 수도 있습니다(예: 보안 또는 추가 인스턴스 작성에 대한 특성). 값을 변경해도 패턴 인스턴스의 작동에 아무런 영향이 없도록 확인해야 합니다.
    • WSDL 파일에 정의된 전송이 HTTPS이고 추가 SSL 구성이 필요하면 WebSphere Message Broker 문서에서 "SSL 인증 구현"을 참조하십시오.
  3. 큐를 작성하십시오. 패턴을 적용하면 큐 이름과 큐 관리자 이름을 포함한 WebSphere MQ 큐 목록이 생성되며, WebSphere MQ 큐가 아직 없으면 WebSphere MQ 큐를 작성해야 합니다. 패턴에 필요한 모든 메시지 설정은 메시지에 명시적으로 설정되므로 큐 설정에 종속되지 않습니다.
  4. 요약 파일에서 작성해야 하는 큐 목록을 확인하십시오. 요약 파일은 InstanceName 패턴 인스턴스 프로젝트의 패턴 구성 폴더에 있습니다.
  5. WebSphere MQ 자원을 작성하십시오. 명령행에서 runmqsc와 같은 유틸리티와 프로그램을 사용하거나 WebSphere MQ 탐색기 그래픽 인터페이스를 사용하여 WebSphere MQ 자원을 작성할 수 있습니다. WebSphere MQ 자원 작성에 대한 자세한 정보는 WebSphere MQ 라이브러리 웹 페이지에 있는 WebSphere MQ Intercommunication을 참조하십시오.
  6. BAR 파일을 배치하십시오. 배치 가능한 WSDL을 포함한 메시지 세트는 동일한 BAR 파일 또는 다른 BAR 파일을 사용하여 배치해야 합니다.

    프록시 서비스의 주소는 배치 후 판별되어 요청자에 공개됩니다. mqsireportproperties 명령으로 실행 그룹에 대한 HTTPCOnnector 특성을 판별하여 이 정보를 구할 수 있습니다.
