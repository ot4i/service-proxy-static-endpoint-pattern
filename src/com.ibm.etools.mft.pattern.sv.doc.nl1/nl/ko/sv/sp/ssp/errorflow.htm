서비스 프록시: 정적 엔드포인트 패턴의 오류 서브플로우

메시지 플로우 처리 중에 오류가 포착되면 오류 서브플로우가 호출되고 요청을 완료하지 못했음을 알리는 오류 응답이 클라이언트로 리턴됩니다.

오류 메시지를 사용하도록 패턴 인스턴스를 구성한 경우, 메시지가 오류 큐에 기록됩니다. 오류 서브플로우에서는 다음과 같은 조치를 수행합니다.

  1. 오류 서브플로우는 서비스 요청자로 리턴되는 SOAP 결함 메시지를 빌드하여 요청이 성공하지 못했음을 알려줍니다. SOAP 결함은 메시지 플로우 오류를 일으킨 예외의 세부사항을 포함합니다.
  2. 오류 메시지를 포함하도록 패턴 매개변수를 구성하고 사용자 정의 특성 ErrorLoggingOn이 선택되면, 오류 메시지를 준비하고 오류 큐에 기록됩니다.

    예외 세부사항이 XML 메시지로 기록됩니다. 해당 세부사항은 다음과 같습니다.

    • 브로커 이름
    • 플로우 이름
    • 시간소인
    • 상태 코드:
      • 1(요청이 전달된 경우)
      • 2(제공자가 응답한 경우)
      • -2(제공자가 결함을 리턴한 경우)
    • 서비스 이름
    • 서비스 네임스페이스
    • 포트
    • 포트 유형
    • 조작
    • 대상 엔드포인트
    • 예외 데이터 요약

    예외 데이터 요약에는 메시지 플로우 레이블, 오류 번호, 텍스트 설명 및 추가 세부사항을 제공하는 삽입 목록이 포함되어 있습니다.
