Tâches à exécuter après avoir généré le modèle Proxy de service : noeud final statique

Tâches que vous devez exécuter après avoir généré le modèle.

  1. Créez le fichier archive de courtier (BAR). Ajoutez le flux de messages généré au fichier BAR. Dans l'éditeur d'archives du courtier, vous devez sélectionner l'option Compiler et intégrer des ressources avant de générer le fichier BAR.
  2. Configurez le fichier archive de courtier. Il n'est pas nécessaire d'effectuer d'autres opérations de configuration au niveau du flux de messages dans le fichier archive de courtier avant de déployer le flux de messages de l'instance du modèle Proxy de service : noeud final statique. Néanmoins, vous pouvez vérifier certaines options par défaut et configurer des modifications pour votre environnement.

    Dans la page Gestion de l'éditeur Archive de courtier :

    • Facultatif : utilisez la propriété RequestLoggingOn définie par l'utilisateur pour indiquer si la consignation est activée ou désactivée pour le flux de messages.
    • Facultatif : utilisez la propriété ResponseLoggingOn définie par l'utilisateur pour indiquer si la consignation est activée ou désactivée pour le flux de messages.
    • Facultatif : utilisez la propriété ErrorLoggingOn définie par l'utilisateur pour indiquer si des messages d'erreur sont générés.
    • Facultatif : vous pouvez également modifier les propriétés configurables dans l'archive de courtier, telles que les propriétés de sécurité ou de création d'instances supplémentaires. Vous devez vérifier que les valeurs que vous changez n'affectent pas le comportement de l'instance du modèle.
    • Si le transport défini dans le fichier WSDL est HTTPS, des opérations de configuration SSL supplémentaires sont nécessaires. Voir "Implémentation de l'authentification SSL" dans la documentation d'IBM Integration.
  3. Créez les files d'attente. Lorsque vous appliquez le modèle, la liste des files d'attente WebSphere MQ est générée, avec les noms de file d'attente et de gestionnaire de files d'attente, que vous devez créer s'ils n'existent pas. Tous les paramètres de message nécessaires au modèle sont définis explicitement dans les messages et ne dépendent pas des paramètres de file d'attente.
  4. Dans le fichier récapitulatif, consultez la liste des files d'attente que vous devez créer. Ce fichier se trouve dans le dossier Configuration du modèle du projet d'instance de modèle InstanceName.
  5. Créez les ressources WebSphere MQ. Vous pouvez créer les ressources WebSphere MQ depuis la ligne de commande en utilisant des programmes et des utilitaires, tels que runmqsc, ou en utilisant l'interface graphique WebSphere MQ Explorer. Pour plus d'informations sur la création des ressources WebSphere MQ, voir WebSphere MQ Intercommunication sur la page Web Bibliothèque WebSphere MQ.
  6. Déployez le fichier BAR. L'ensemble de messages qui contient le fichier déployable WSDL doit être déployé en utilisant le même ou un autre fichier BAR.

    L'adresse du service proxy est déterminée après le déploiement et publiée vers les demandeurs. Les informations peuvent être obtenues en utilisant la commande mqsireportproperties pour déterminer les propriétés HTTPCOnnector du groupe d'exécution.
