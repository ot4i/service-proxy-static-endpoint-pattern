Contraintes d'utilisation du modèle Proxy de service : noeud final statique

Il existe certaines contraintes lorsque vous utilisez ce modèle.

Vous pouvez utiliser le modèle Proxy de service : noeud final statique uniquement si toutes les contraintes suivantes sont respectées :

  • Les demandes doivent être reçues sous forme de messages SOAP sur HTTP ou HTTPS.
  • Les applications demandeurs doivent prendre en charge un style d'interaction réponse-demande synchrone.
  • Le service cible doit fournir une liaison SOAP sur HTTP ou HTTPS.
  • Les fichiers WSDL qui définissent l'interface du service cible peuvent avoir plusieurs types de ports et de liaisons, mais seul le premier type de port ou de liaison est utilisé pour configurer l'instance de modèle générée. Vous pouvez sélectionner des alternatives dans l'éditeur de flux de messages.
  • Si la consignation est nécessaire, le format utilisé par le modèle doit répondre aux conditions de votre environnement.
