Flux secondaire Journal pour le modèle Proxy de service : noeud final statique

Lorsque la consignation est activée, le flux secondaire Journal consigne les demandes et les réponses en fonction des propriétés RequestLoggingOn et ResponseLoggingOn définies par l'utilisateur.

Les messages de journal sont écrits comme messages persistants WebSphere MQ dans une file d'attente WebSphere MQ. Ces messages de journal sont ensuite traités séparément conformément aux pratiques standard de votre entreprise. Le journal des messages contient le message d'origine et les informations suivantes dans un en-tête MQRFH2 :

  • Horodatage
  • Nom du courtier
  • Nom du flux de messages
  • Détails du service
  • Etat (1 = demande, 2 = réponse, -2 = erreur)
