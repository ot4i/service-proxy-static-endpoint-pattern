Flux secondaire Erreur pour le modèle Proxy de service : noeud final statique

Si des erreurs sont interceptées au cours du traitement du flux de messages, le flux secondaire Erreur est appelé et un message d'erreur est envoyé au client pour indiquer que la demande n'a pas abouti.

Si vous configurez l'instance du modèle pour activer les messages d'erreur, un message est écrit dans la file d'attente des erreurs. Le flux secondaire Erreur exécute les actions suivantes :

  1. Il crée un message d'erreur SOAP qui est envoyé au demandeur de service pour indiquer que la demande n'a pas abouti. L'erreur SOAP non spécifiée inclut les détails de l'exception qui a provoqué l'échec du flux de messages.
  2. Si vous avez configuré les paramètres du modèle pour inclure des messages d'erreur et que la propriété définie par l'utilisateur, ErrorLoggingOn est sélectionnée, un message d'erreur est rédigé et s'affiche dans la file d'attente des erreurs.

    Les détails de l'exception sont écrits sous forme de message XML. Ces détails comprennent :

    • Nom du courtier
    • Nom du flux
    • Horodatage
    • Code d'état :
      • 1 (lorsque la demande a été transmise)
      • 2 (si le fournisseur a répondu)
      • -2 (si le fournisseur renvoie une erreur)
    • Nom de service
    • Espace de nom de service
    • Port
    • Type de port
    • Opération
    • Noeud final cible
    • Récapitulatif des données de l'exception

    Le récapitulatif des données d'exception contient le libellé du flux de messages, le numéro d'erreur, la description de texte, ainsi qu'une liste d'insertions donnant plus de détails.
