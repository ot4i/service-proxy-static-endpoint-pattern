Informations sur le flux Proxy du modèle Proxy de service : noeud final statique.

Le flux SimpleServiceProxy traite toutes les demandes de service de la manière suivante :

  1. Lorsque le flux de messages reçoit une demande, la demande est validée si la configuration de l'instance du modèle impose la validation.
  2. Le flux de messages définit les paramètres nécessaires pour envoyer la demande de service au fournisseur de service cible en utilisant l'URL de fournisseur fournie dans un paramètre du modèle ou dans le fichier archive de courtier (BAR).
  3. La demande SOAP est ensuite envoyée au fournisseur du service. Si une réponse est reçue, elle est renvoyée directement au demandeur de service d'origine. Si un message d'erreur est reçu, l'état du flux est défini pour indiquer qu'une erreur s'est produite et le message d'erreur est retourné au demandeur du service.
  4. Vous pouvez configurer le modèle pour inclure ou exclure la consignation des demandes et des réponses.
  5. Si la consignation est exclue, le flux de messages se termine correctement lorsque la réponse est retournée.
  6. Le flux de messages contient également un flux secondaire pour traiter les erreurs.
  7. Les messages d'erreur sont écrits dans la file d'attente des erreurs si vous avez défini cette option pour l'instance du modèle.
