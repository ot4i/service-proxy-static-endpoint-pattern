Tâches à exécuter avant d'appliquer le modèle Proxy de service : noeud final statique

Avant d'appliquer le modèle pour produire une instance de modèle, vous devez déterminer les tâches à exécuter parmi les tâches suivantes. Si nécessaire, exécutez les tâches suivantes :

Si nécessaire, exécutez les tâches suivantes :

  • Vérifiez que le modèle répond aux besoins de l'entreprise et qu'il peut être pris en charge par les applications demandeurs et fournisseurs.
  • Définissez l'interface du service cible.
  • Nommez l'instance du modèle. Identifiez la fonction de l'application de modèle pour déterminer le nom approprié.
  • Déterminez si vous devez utiliser la consignation. Si vous devez l'utiliser, vous devez déterminer le gestionnaire de files d'attente et la file d'attente auxquels les messages de consignation sont envoyés. La consignation doit être exécutée dans une file d'attente de consignation générale qui peut déjà exister pour d'autres flux de courtier. Vous pouvez effectuer la consignation dans une file d'attente créée pour l'instance de modèle à créer.
  • Déterminez si vous avez besoin des messages d'erreur.
  • Déterminez si vous devez utiliser la validation.
  • Créez un fichier déployable WSDL dans un projet d'ensemble de messages. Consultez la rubrique "Générer WSDL" dans la documentation d'IBM Integration.
