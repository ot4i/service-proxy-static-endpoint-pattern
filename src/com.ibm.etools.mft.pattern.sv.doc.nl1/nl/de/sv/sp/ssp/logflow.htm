Untergeordneter Protokollnachrichtenfluss für das Muster Service Proxy: static endpoint (Service-Proxy: statischer Endpunkt)

Wenn die Protokollierung aktiviert ist, zeichnet der untergeordnete Protokollnachrichtenfluss Anforderungen und Antworten gemäß der benutzerdefinierten Eigenschaften RequestLoggingOn und ResponseLoggingOn auf.

Protokollnachrichten werden als persistente WebSphere MQ-Nachrichten in eine WebSphere MQ-Warteschlange geschrieben. Diese Protokollnachrichten werden unabhängig voneinander gemäß der in Ihrem Unternehmen implementierten Standardverfahren verarbeitet. Das Nachrichtenprotokoll enthält die ursprüngliche Nachricht sowie die folgenden Informationen in einem MQRFH2-Header:

  • Zeitmarke
  • Brokername
  • Nachrichtenflussname
  • Servicedetails
  • Status (1 = Anforderung, 2 = Antwort, -2 = Fehler)
