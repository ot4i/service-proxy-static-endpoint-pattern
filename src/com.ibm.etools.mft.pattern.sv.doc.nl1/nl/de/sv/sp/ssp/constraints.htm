Vorgaben für die Verwendung des Musters Service Proxy: static endpoint (Service-Proxy: statischer Endpunkt)

Wenn Sie dieses Muster verwenden, gelten eine Reihe von Vorgaben.

Das Muster Service Proxy: static endpoint (Service-Proxy: statischer Endpunkt) kann nur verwendet werden, wenn die folgenden Bedingungen erfüllt sind:

  • Die Anforderungen müssen als SOAP-Nachrichten über HTTP oder HTTPS eingehen.
  • Die anfordernden Anwendungen müssen einen Interaktionsstil mit synchronen Anforderungen und Antworten unterstützen.
  • Der Zielservice muss die Bindung SOAP über HTTP oder HTTPS bereitstellen.
  • Die WSDLs mit der Definition der Zielserviceschnittstelle können mehrere Porttypen und Bindungen enthalten; zur Konfiguration der generierten Musterinstanz wird aber nur der erste Porttyp bzw. die erste Bindung verwendet. Alternativen können im Nachrichtenflusseditor ausgewählt werden.
  • Wenn eine Protokollierung erforderlich ist, wird davon ausgegangen, dass das von diesem Muster verwendete Format den Anforderungen Ihres Unternehmens entspricht.
