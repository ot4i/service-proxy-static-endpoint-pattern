Erforderliche Tasks vor der Anwendung des Musters Service Proxy: static endpoint (Service-Proxy: statischer Endpunkt)

Bevor Sie das Muster für die Erstellung einer Musterinstanz anwenden, müssen Sie bestimmen, welche der folgenden Tasks erforderlich sind. Führen Sie folgende Tasks aus, falls erforderlich:

Führen Sie die erforderlichen Tasks bei Bedarf aus:

  • Prüfen Sie, ob dieses Muster Ihren Geschäftsanforderungen entspricht und sowohl von den Requester- als auch von den Provideranwendungen unterstützt wird.
  • Definieren Sie die Zielserviceschnittstelle.
  • Wählen Sie einen Namen für die Musterinstanz aus. Bestimmen Sie den Zweck der Musteranwendung, sodass Sie einen geeigneten Namen festlegen können.
  • Entscheiden Sie, ob die Protokollierung erforderlich ist. Falls ja, bestimmen Sie den Warteschlangenmanager und die Warteschlange für die Protokollnachrichten. Die Protokollierung muss mit einer allgemeinen Protokollierungswarteschlange stattfinden, die möglicherweise bereits für andere Brokernachrichtenflüsse vorhanden ist. Die Protokollierung kann aber auch in eine Warteschlange erfolgen, die Sie speziell für die neue Musterinstanz erstellen.
  • Entscheiden Sie, ob Fehlernachrichten erforderlich sind.
  • Entscheiden Sie, ob die Validierung erforderlich ist.
  • Erstellen Sie den implementierbaren WSDL-Code in einem Nachrichtengruppenprojekt (siehe "Generate WSDL" (WSDL generieren) in der IBM Integration-Dokumentation).
