Service Proxy: static endpoint 型樣

使用此型樣，透過直接連結至目標服務提供者的虛擬服務進行遞送，以提供 Web 服務要求者與 Web 服務提供者之間的取消聯結功能。

使用此型樣的時機如下：

  • 因為服務所在端點的位址可能會變更，或必須將它隱藏以控制服務的存取，所以您不想要使用戶端直接存取服務時
  • 若要支援透過 HTTP 或 HTTPS 的服務存取
  • 當您想要提供服務提供者的所有要求記載功能時，或者想要提供其他錯誤處理功能而不想要變更服務提供者時。

解決方案

解決方案是實作訊息流程，而此訊息流程會提供提供者 Web 服務的 Proxy，也可以包括記載及錯誤處理功能。
