產生 Service Proxy: static endpoint 型樣之後要完成的作業

在您產生此型樣之後必須完成的作業。

  1. 建立分配管理系統保存檔 (BAR)。將產生的訊息流程新增至 BAR 檔。 在「分配管理系統保存檔」編輯器中，您必須先選取編譯及行內資源選項，然後再建置 BAR 檔。
  2. 配置分配管理系統保存檔。在您部署 Service Proxy: static endpoint 型樣實例的訊息流程之前，並不需要進一步配置分配管理系統保存檔中的訊息流程，但是您可能會想要檢閱一些預設選項，以及配置環境的變更。

    在分配管理系統保存檔編輯器的管理頁面中，可執行下列動作：

    • 選用項目：使用 RequestLoggingOn 使用者定義的內容，以選取是要開啟還是關閉訊息流程的記載要求。
    • 選用項目：使用 ResponseLoggingOn 使用者定義的內容，以選取是要開啟還是關閉訊息流程的記載回覆。
    • 選用項目：使用 ErrorLoggingOn 使用者定義的內容，以選取是否產生錯誤訊息。
    • 選用項目：您也可以變更分配管理系統保存檔中的任何可配置內容，例如，安全內容或建立其他實例的內容。您必須確保所變更的任何值不會影響型樣實例的行為。
    • 如果 WSDL 檔案中定義的傳輸是 HTTPS，則需要其他 SSL 配置，請參閱 IBM Integration 說明文件中的「實作 SSL 鑑別」。
  3. 建立佇列。當您套用型樣時，會產生具有佇列名稱及佇列管理程式名稱的 WebSphere MQ 佇列清單，如果這些 WebSphere MQ 佇列尚未存在，則必須加以建立。此型樣需要的所有訊息設定都是明確設定於訊息上，而且與佇列設定無關。
  4. 在摘要檔中，檢視您必須建立的佇列清單。摘要檔位於 InstanceName 型樣實例專案的「型樣配置」資料夾中。
  5. 建立 WebSphere MQ 資源。您可以在指令行上使用程式及公用程式（例如 runmqsc），或使用「WebSphere MQ 探險家」圖形介面，以建立 WebSphere MQ 資源。 如需建立 WebSphere MQ 資源的相關資訊，請參閱位於 WebSphere MQ Library 網頁上的 WebSphere MQ Intercommunication。
  6. 部署 BAR 檔案。必須使用相同或不同的 BAR 檔案，部署含有可部署 WSDL 的訊息集。

    Proxy 服務的位址是在部署之後判定，然後發佈給要求者。使用 mqsireportproperties 指令可以取得此資訊，以判斷執行群組的 HTTPCOnnector 內容。
