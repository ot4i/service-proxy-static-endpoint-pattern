使用 Service Proxy: static endpoint 型樣的限制

當您使用此型樣時，將適用一些限制。

只有在可以符合下列所有限制時，才能使用 Service Proxy: static endpoint 型樣：

  • 要求必須透過 HTTP 或 HTTPS，以 SOAP 訊息形式接收。
  • 要求端應用程式必須支援同步要求回應互動樣式。
  • 目標服務必須透過 HTTP 或 HTTPS 連結提供 SOAP。
  • 定義目標服務介面的 WSDL 可以具有多個埠類型及連結，但是只會使用第一個埠類型或連結來配置產生的型樣實例。您可以在訊息流程編輯器中選取替代項目。
  • 如果需要記載，則會採用此型樣所使用的格式，以符合您自己環境的需求。
