Service Proxy: static endpoint 型樣的 Proxy 流程

Service Proxy: static endpoint 型樣的 Proxy 流程明細。

SimpleServiceProxy 流程使用下列方法來處理所有服務要求：

  1. 訊息流程接收到要求時，會驗證要求（如果型樣實例配置需要驗證）。
  2. 訊息流程設定必要的參數，透過型樣參數中所提供的提供者 URL 或分配管理系統保存檔 (BAR) 中配置的提供者 URL，以將服務要求轉遞給目標服務提供者。
  3. 然後，會將 SOAP 要求傳送至服務提供者。如果接收到順利完成的回應，則會將此回應直接傳回給起始端服務要求者。 如果接收到錯誤訊息，則會設定流程狀態以指出發生錯誤，並將錯誤訊息傳回至服務要求者。
  4. 您可以配置型樣以包括或排除要求及回覆的記載。
  5. 如果不包括記載，則訊息流程會在傳回回覆時順利結束。
  6. 訊息流程也包含錯誤處理的子流程。
  7. 如果您已針對型樣實例配置此選項，則會將錯誤訊息寫入錯誤佇列。
