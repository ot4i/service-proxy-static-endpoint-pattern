套用 Service Proxy: static endpoint 型樣之前要完成的作業

套用型樣以產生型樣實例之前，您必須決定需要下列哪一項作業。必要的話，請完成下列作業：

必要的話，請完成必要的作業：

  • 識別此型樣是否符合商業需求，而且要求者及提供者應用程式是否都支援此型樣。
  • 定義目標服務介面。
  • 命名型樣實例。識別型樣應用程式的用途，讓您可以決定適當的名稱。
  • 決定您是否需要記載。如果需要記載，則您必須決定傳送記載訊息的目標佇列管理程式及佇列。 記載作業必須記載至其他分配管理系統流程的現有一般日誌佇列。記載作業可能是記載至針對想要建立的型樣實例所建立的佇列。
  • 決定您是否需要錯誤訊息。
  • 決定您是否需要驗證。
  • 在訊息集專案中建立可部署的 WSDL，請參閱 IBM Integration 說明文件中的「產生 WSDL」。
