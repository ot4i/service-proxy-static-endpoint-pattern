Service Proxy: static endpoint 型樣的錯誤子流程

如果在訊息流程處理期間捕捉到錯誤，則會呼叫「錯誤」子流程，並將錯誤回應傳回給用戶端，指出要求未順利完成。

如果您已配置型樣實例以啟用錯誤訊息，則會在錯誤佇列中寫入訊息。「錯誤」子流程會採取下列動作：

  1. 「錯誤」子流程會建置傳回給服務要求者的 SOAP 錯誤訊息，指出要求未順利完成。SOAP 錯誤包括導致訊息流程失敗的異常狀況明細。
  2. 如果您已將型樣參數配置成包括錯誤訊息，並選取使用者定義的內容 ErrorLoggingOn，則會準備錯誤訊息，並將它寫入錯誤佇列。

    異常狀況的明細會寫入為 XML 訊息。這些明細包括：

    • 分配管理系統名稱
    • 流程名稱
    • 時間戳記
    • 狀態碼：
      • 1（已轉遞要求時）
      • 2（如果回覆提供者）
      • -2（如果提供者傳回錯誤）
    • 服務名稱
    • 服務名稱空間
    • 埠
    • 埠類型
    • 作業
    • 目標端點
    • 異常狀況資料的摘要

    異常狀況資料的摘要包含訊息流程標籤、錯誤碼、文字說明，以及提供進一步明細的插入項目清單。
