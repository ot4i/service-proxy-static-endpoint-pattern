Service Proxy: static endpoint 型樣的日誌子流程

啟用記載時，「日誌」子流程會根據使用者定義的內容 RequestLoggingOn 及 ResponseLoggingOn，來記載要求及回覆。

日誌訊息會以持續 WebSphere MQ 訊息形式寫入 WebSphere MQ 佇列。會根據組織內的標準實務，個別處理這些日誌訊息。訊息日誌會在 MQRFH2 標頭中包含原始訊息及下列資訊：

  • 時間戳記
  • 分配管理系統名稱
  • 訊息流程名稱
  • 服務明細
  • 狀態（1 = 要求、2 = 回覆、-2 = 錯誤）
