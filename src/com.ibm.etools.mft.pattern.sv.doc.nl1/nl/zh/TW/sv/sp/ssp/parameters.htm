Service Proxy: static endpoint 型樣的參數

服務資訊
型樣參數 M C 預設值 說明
服務提供者的 WSDL 是 否  

此參數定義可部署的 WSDL 檔案，而此檔案定義建立其 Proxy 的服務介面。

可部署的 WSDL 檔案必須從工作區訊息集中進行選取。
服務提供者的 URL 否 是  

此參數定義進行 Proxy 處理的服務所在的端點。

如果未指定，則會根據 WSDL 檔案中的連結資訊來決定端點。
Proxy URL 否 是  

此參數定義 URL 路徑，以用來定義已部署 Proxy 的端點。

完整服務位址也包括主機名稱及埠（在部署 Proxy 時決定）。

如果未指定任何路徑，則會使用來自服務 WSDL 的路徑。
SOAP 要求的驗證 否 是 無

送入的 SOAP 訊息會根據此參數的定義進行驗證。

有效值為：

  • 無
  • 內容和值
SOAP 回應的驗證 否 是 無

SOAP 回應會根據此參數的定義進行驗證。

有效值為：

  • 無
  • 內容和值
記載
型樣參數 M C 預設值 說明
需要記載 否 否 已清除

此型樣參數決定型樣實例是否包括用於記載的程式碼。

如果清除需要記載，則不會包括「日誌」子流程。

如果選取需要記載，則「日誌」子流程會併入訊息流程中，而且必須設定日誌佇列管理程式及日誌佇列的值。
日誌佇列管理程式 否 是

此型樣參數定義佇列管理程式以進行記載。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。

只有在選取需要記載時，才需要此參數。

日誌佇列 否 是 LOG

此型樣參數定義佇列以進行記載。

只有在選取需要記載時，才需要此參數。
錯誤處理
型樣參數 M C 預設值 說明
需要錯誤訊息 否 否 已選取

此型樣參數定義型樣應用程式是否建立可建立錯誤訊息的訊息流程元素。

如果選取需要錯誤訊息，則必須設定錯誤佇列管理程式及錯誤佇列的值。
錯誤佇列管理程式 否 是  

此型樣參數定義錯誤訊息的佇列管理程式。 如果使用分配管理系統佇列管理程式以進行記載，則可以將它保留為空白。
錯誤佇列 否 是 ERROR

此型樣參數定義錯誤訊息的佇列。

只有在選取需要錯誤訊息時，才需要此參數。

一般
型樣參數 M C 預設值 說明
分配管理系統綱目 否 否 mqsi

如果定義分配管理系統綱目，則會在該綱目內建立訊息流程及 ESQL 檔案。 如果未定義任何綱目，則會使用預設分配管理系統綱目。
流程字首 否 否  

如果已設定，則會將此字首新增至所有訊息流程名稱的開頭。

您只可以使用訊息流程名稱中的有效字元。
流程字尾 否 否  

如果已設定，則會將此字尾新增至所有訊息流程名稱的結尾。

您只可以使用訊息流程名稱中的有效字元。
佇列字首 否 否  

如果已設定，則會將此字首新增至所有佇列名稱的開頭。

如果使用預設佇列名稱，則必須設定字首及（或）字尾，才能識別每一個型樣實例的佇列名稱。

佇列字尾 否 否  

如果已設定，則會將此字尾新增至所有佇列名稱的結尾。

如果使用預設佇列名稱，則必須設定字首及（或）字尾，才能識別每一個型樣實例的佇列名稱。

簡要說明 否 否  

此說明會新增至所有產生的訊息流程的簡要說明內容。
詳細說明 否 否  

此說明會新增至所有產生的訊息流程的詳細說明內容。

所有型樣參數會與其預設值一起列出。

標頭為 M 的直欄指出內容是否為必要的。 標頭為 C 的直欄指出內容是否為可配置的，這表示您可以在將訊息流程新增至分配管理系統保存檔 (BAR) 時變更值。

如果未指出任何預設值，則預設值是空字串。
