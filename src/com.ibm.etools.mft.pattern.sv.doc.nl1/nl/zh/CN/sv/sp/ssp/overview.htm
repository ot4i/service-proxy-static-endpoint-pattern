Service Proxy: static endpoint 模式

使用此模式通过直接绑定至目标服务提供者的虚拟服务进行路由，从而在 Web Service 请求者和 Web Service 提供者之间提供解耦。

在以下情况下使用此模式：

  • 在不希望客户机直接访问服务时，这是因为服务所在端点的地址可能会发生更改，或者因为其必须隐藏以控制对服务的访问
  • 要支持通过 HTTP 或 HTTPS 访问服务
  • 在您希望为服务提供者提供所有请求记录时，或者希望提供其他错误处理功能而不对服务提供者进行更改时

解决方案

解决方案将实现一个消息流，该流为提供者 Web Service 提供代理，并且还包含记录和错误处理功能。
