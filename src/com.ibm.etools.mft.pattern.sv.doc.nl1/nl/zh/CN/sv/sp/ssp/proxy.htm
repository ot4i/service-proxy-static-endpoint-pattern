Service Proxy: static endpoint 模式的代理流

有关 Service Proxy: static endpoint 模式的代理流的详细信息。

SimpleServiceProxy 流通过以下方式处理所有服务请求：

  1. 如果模式实例配置要求验证，那么在消息流接收请求时，将验证请求。
  2. 消息流通过使用模式参数提供的或者在代理归档 (BAR) 文件中配置的提供者 URL，设置将服务请求转发至目标服务提供者所需的参数。
  3. 然后将 SOAP 请求发送至服务提供者。 如果收到成功响应，那么此响应将直接返回至初始服务请求者。 如果接收到故障消息，那么将设置流状态以指示发生故障，并将故障消息返回至服务请求者。
  4. 您可以配置模式以包含或排除请求和应答记录。
  5. 如果不包含记录，那么在返回应答时消息流成功结束。
  6. 消息流还包含用于错误处理的子流。
  7. 如果已为模式实例配置了此选项，那么会将错误消息写入到错误队列。
