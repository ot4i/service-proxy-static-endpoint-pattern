生成 Service Proxy: static endpoint 模式之后要完成的任务

生成该模式之后必须完成的任务。

  1. 创建代理归档 (BAR) 文件。将生成的消息流添加到 BAR 文件中。在“代理归档”编辑器中，必须先选中编译并内联资源选项，然后再构建 BAR 文件。
  2. 配置代理归档文件。为 Service Proxy: static endpoint 模式实例部署消息流之前，不需要在代理归档文件中进一步配置消息流，但应当针对您的环境复审一些缺省选项并配置更改。

    在“代理归档”编辑器的“管理”页面中：

    • 可选：使用用户定义的属性 RequestLoggingOn 来选择是要为消息流开启还是关闭记录请求。
    • 可选：使用用户定义的属性 ResponseLoggingOn 来选择是要为消息流开启还是关闭记录应答。
    • 可选：使用用户定义的属性 ErrorLoggingOn 来选择是否生成错误消息。
    • 可选：您还可以更改代理归档中的任何可配置属性，例如，安全性属性或用于创建其他实例的属性。您必须确保更改的任何值都不会影响模式实例的行为。
    • 如果在 WSDL 文件中定义的传输方法是 HTTPS，那么还需要其他的 SSL 配置，请参阅 IBM Integration 文档中的“实现 SSL 认证”。
  3. 创建队列。应用模式时，将生成一个 WebSphere MQ 队列的列表，其中会列出队列名称和队列管理器名称；如果不存在这些 WebSphere MQ 队列，就必须加以创建。模式所需的所有消息设置都必须明确设置在消息上，且独立于队列的设置。
  4. 在摘要文件中，查看您必须创建的队列列表。摘要文件位于 InstanceName 模式实例项目的模式配置文件夹中。
  5. 创建 WebSphere MQ 资源。您可以使用程序和实用程序（如 runmqsc）在命令行上创建 WebSphere MQ 资源，也可以使用 WebSphere MQ Explorer 图形界面来完成创建。有关创建 WebSphere MQ 资源的更多信息，请参阅 WebSphere MQ Library Web 页面上提供的 WebSphere MQ Intercommunication。
  6. 部署 BAR 文件。必须使用同一个或其他 BAR 文件来部署包含可部署 WSDL 的消息集。

    代理服务的地址在部署后确定并发布给请求者。获取此信息的方法是：使用 mqsireportproperties 命令来确定执行组的 HTTPCOnnector 属性。
