Tareas que deben completarse antes de aplicar el patrón Proxy de servicio: punto final estático

Antes de aplicar el patrón para producir una instancia de patrón, debe determinar cuáles de las tareas siguientes son necesarias. Si es preciso, realice las tareas siguientes:

Si es preciso, realice las tareas necesarias:

  • Identificar que este patrón cumple el requisito empresarial y las aplicaciones de solicitante y proveedor pueden darle soporte.
  • Definir la interfaz de servicios de destino.
  • Asignar un nombre a la instancia de patrón. Identifique el fin de la aplicación de patrón para que pueda decidir cuál es el nombre adecuado.
  • Decidir si necesita un registro. Si el registro es necesario, debe decidir a qué gestor de colas y cola se enviarán los mensajes de registro. El registro debe efectuarse en una cola de registro general, que ya puede existir para otros flujos de intermediario. Es posible que se realice el registro en una cola creada por la instancia de patrón que desea crear.
  • Decidir si necesita mensajes de error.
  • Decidir si necesita validación.
  • Crear WSDL desplegable en un proyecto de conjunto de mensajes; consulte "Generar WSDL" en la documentación de IBM Integration.
