Patrón Proxy de servicio: punto final estático

Utilice este patrón para proporcionar desacoplamiento entre solicitantes de servicio web y proveedores de servicio web direccionándolo a través de un servicio virtual enlazado directamente al proveedor de servicio de destino.

Utilice este patrón:

  • Cuando no desee que el cliente acceda al servicio directamente, porque la dirección del punto final en el que está ubicado el servicio puede cambiar o porque debe estar oculto para controlar el acceso al servicio
  • Para dar soporte al acceso al servicio a través de HTTP o HTTPS
  • Cuando se desea proporcionar registro de todas las solicitudes al proveedor de servicio, o cuando se desea proporcionar prestaciones adicionales de manejo de errores sin realizar cambios en el proveedor de servicio

Solución

La solución es implementar un flujo de mensajes que proporcione un proxy para el servicio web de proveedor, y también puede incluir prestaciones de registro y manejo de errores.
