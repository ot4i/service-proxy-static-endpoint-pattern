Restricciones en el uso del patrón Proxy de servicio: punto final estático

Se aplican una serie de restricciones cuando se utiliza este patrón.

Sólo puede utilizar el patrón Proxy de servicio: punto final estático si puede cumplir todas las restricciones siguientes:

  • Las solicitudes deben recibirse como mensajes SOAP a través de HTTP o HTTPS.
  • Las aplicaciones solicitantes deben dar soporte a un estilo de interacción de respuesta a solicitud síncrona.
  • El servicio de destino debe proporcionar un enlace SOAP a través de HTTP o HTTPS.
  • Los WSDL que definen la interfaz de servicio de destino pueden tener varios tipos de puerto y enlaces, pero sólo el primer tipo de puerto o enlace se utiliza para configurar la instancia de patrón generada. Puede seleccionar alternativas en el editor de flujo de mensajes.
  • Si el registro es necesario, se supone que el formato utilizado por este patrón cumple los requisitos de su propio entorno.
