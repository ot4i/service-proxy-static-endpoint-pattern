Detalles sobre el flujo Proxy para el patrón Proxy de servicio: punto final estático.

El flujo SimpleServiceProxy procesa todas las solicitudes de servicio de la siguiente manera:

  1. Cuando el flujo de mensajes recibe una solicitud, ésta se valida si la configuración de la instancia de patrón necesita validación.
  2. El flujo de mensajes establece los parámetros que son necesarios para reenviar la solicitud e servicio al proveedor de servicio de destino utilizando el URL de proveedor que se ha proporcionado en un parámetro de patrón o se ha configurado en el archivo archivador de intermediario (BAR).
  3. A continuación, la solicitud SOAP se envía al proveedor de servicio. Si se recibe una respuesta satisfactoria, esta respuesta se devuelve directamente al solicitante de servicio que la inició. Si se recibe un mensaje de error, el estado del flujo se establece para indicar que se ha producido un error y el mensaje de error se devuelve al solicitante de servicio.
  4. Puede configurar el patrón para incluir o excluir el registro de solicitudes y respuestas.
  5. Si el registro no está incluido, el flujo de mensajes finaliza correctamente cuando se devuelve la respuesta.
  6. El flujo de mensajes también contiene un subflujo de manejo de errores.
  7. Los mensajes de error se escriben en la cola de errores si ha configurado esta opción para la instancia de patrón.
