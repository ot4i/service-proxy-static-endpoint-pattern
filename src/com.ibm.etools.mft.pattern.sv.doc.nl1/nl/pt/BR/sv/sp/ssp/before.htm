Tarefas para Concluir Antes de Aplicar o Padrão de Service Proxy: Terminal Estático

Antes de aplicar o padrão para produzir uma instância padrão, é necessário determinar quais das tarefas a seguir são necessárias. Se necessário, conclua as tarefas a seguir:

Se necessário, conclua as tarefas necessárias:

  • Identifique se este padrão atende ao requisito de negócios e pode ser suportado pelos aplicativos do solicitante e do provedor.
  • Defina a interface de serviço de destino.
  • Nomeie a instância padrão. Identifique o propósito do aplicativo padrão para que você possa decidir sobre um nome apropriado.
  • Decida se você requer a criação de log. Se a criação de log for necessária, é preciso decidir para qual gerenciador de filas e fila as mensagens de criação de log são enviadas. A criação de log deve ser para uma fila de logs gerais, que já podem existir para outros fluxos do broker. A criação de log pode ser para uma fila que é criada para a instância padrão que você deseja criar.
  • Decida se você requer mensagens de erro.
  • Decida se você requer validação.
  • Crie o WSDL implementável em um projeto de conjunto de mensagens, consulte "Gerar WSDL" na documentação do IBM Integration.
