Tarefas para Concluir Após Gerar o Padrão de Service Proxy: Terminal Estático

As tarefas que você deve concluir após ter gerado o padrão.

  1. Crie o arquivo broker archive (BAR). Inclua o fluxo de mensagens gerado em seu arquivo BAR. No editor do Broker Archive, você deve selecionar a opção Compilar e Sequenciar Recursos antes de construir o arquivo BAR.
  2. Configure o arquivo broker archive. A configuração adicional do fluxo de mensagens no arquivo bar não é necessária antes de implementar os fluxos de mensagens para sua instância padrão de Service Proxy: Terminal Estático, mas talvez você deseje revisar algumas das opções padrão e configurar alterações para seu ambiente.

    Na página Gerenciar do editor de Broker Archive:

    • Opcional: Use a propriedade definida pelo usuário RequestLoggingOn para selecionar se os pedidos de criação de log estão ativados ou desativados para o fluxo de mensagens.
    • Opcional: Use a propriedade definida pelo usuário ResponseLoggingOn para selecionar se as respostas de criação de log estão ativadas ou desativadas para o fluxo de mensagens.
    • Opcional: Use a propriedade definida pelo usuário ErrorLoggingOn para selecionar se as mensagens de erro são geradas.
    • Opcional: Também é possível alterar qualquer uma das propriedades configuráveis no broker archive, por exemplo, propriedades para segurança ou criar instâncias adicionais. É necessário assegurar que quaisquer valores que você alterar não causem impacto no comportamento da instância padrão.
    • Se o transporte definido no arquivo WSDL for HTTPS, a configuração de SSL adicional será necessária. Consulte "Implementando a Autenticação SSL" na documentação do IBM Integration.
  3. Crie filas. Quando aplicar o padrão, uma lista de filas do WebSphere MQ será gerada com nomes de filas e nomes de gerenciador de filas, estas filas do WebSphere MQ deverão ser criadas se ainda não existirem. Todas as configurações de mensagem que são necessárias por padrão são configuradas explicitamente nas mensagens e não são dependentes das configurações de filas.
  4. No arquivo de resumo, visualize a lista de filas que você deve criar. O arquivo de resumo está localizado na pasta Configuração de Padrão do projeto de instância padrão InstanceName.
  5. Crie recursos do WebSphere MQ. É possível criar recursos do WebSphere MQ na linha de comandos, usando programas e utilitários como runmqsc, ou usando a interface gráfica do WebSphere MQ Explorer. Para obter informações adicionais sobre a criação de recursos do WebSphere MQ, consulte WebSphere MQ Intercommunication, disponível na página da Web da biblioteca do WebSphere MQ.
  6. Implemente o arquivo BAR. O conjunto de mensagens contendo o WSDL implementável deve ser implementado usando o mesmo arquivo BAR ou outro.

    O endereço do serviço de proxy é determinado após a implementação e é publicado para os solicitantes. Estas informações podem ser obtidas usando o comando mqsireportproperties para determinar as propriedades HTTPCOnnector para o grupo de execução.
