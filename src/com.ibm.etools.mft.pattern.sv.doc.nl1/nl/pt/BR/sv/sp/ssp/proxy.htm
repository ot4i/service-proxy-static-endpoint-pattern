Fluxo do Proxy para o Padrão de Service Proxy: Terminal Estático

Detalhes sobre o fluxo do proxy para o padrão de Service Proxy: Terminal Estático.

O fluxo SimpleServiceProxy processa todos os pedidos de serviço da seguinte maneira:

  1. Quando um pedido é recebido pelo fluxo de mensagens o pedido é validado se a validação é requerida pela configuração de instância padrão.
  2. O fluxo de mensagens configura os parâmetros que são necessários para encaminhar o pedido de serviço ao provedor de serviços de destino usando a URL do provedor que é fornecida em um parâmetro padrão ou configurada no arquivo broker archive (BAR).
  3. O pedido SOAP é, então, enviado ao provedor de serviços. Se uma resposta bem-sucedida é recebida, esta resposta é retornada diretamente ao solicitante de serviços inicial. Se uma mensagem de falha for recebida, o status do fluxo será configurado para indicar que ocorreu uma falha e a mensagem de falha é retornada ao solicitante de serviços.
  4. É possível configurar o padrão para incluir ou excluir a criação de log de pedidos e respostas.
  5. Se a criação de log não for incluída, o fluxo de mensagens terminará com êxito quando a resposta for retornada.
  6. O fluxo de mensagens também contém um subfluxo para manipulação de erros.
  7. As mensagens de erro são gravadas na fila de erros se você configurou esta opção para sua instância padrão.
