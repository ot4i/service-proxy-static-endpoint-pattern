Restrições no Uso do Padrão Service Proxy: Terminal Estático

Várias restrições se aplicam quando este padrão é usado.

É possível usar o padrão Service Proxy: Terminal Estático apenas se você puder atender todas as restrições a seguir:

  • Pedidos devem ser recebidos como mensagens SOAP sobre HTTP ou HTTPS.
  • Aplicativos solicitantes devem suportar um estilo de interação de resposta de pedido síncrono.
  • O serviço de destino deve fornecer uma ligação SOAP sobre HTTP ou HTTPS.
  • WSDLs que definem a interface de serviço de destino podem ter múltiplos tipos de porta e ligações, mas apenas o primeiro tipo de porta ou ligação é usado para configuração da instância padrão gerada. É possível selecionar alternativas no editor de fluxo de mensagens.
  • Se a criação de log for necessária, o formato que é usado por este padrão é assumido para atender aos requisitos de seu próprio ambiente.
