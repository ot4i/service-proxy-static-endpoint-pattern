Padrão de Service Proxy: Terminal Estático

Use esse padrão para fornecer desacoplamento entre solicitantes de serviços da Web e provedores de serviços da Web, roteando através de um serviço virtual que está ligado diretamente ao provedor de serviços de destino.

Use este padrão:

  • Quando não desejar que o cliente acesse o serviço diretamente, porque o endereço do terminal no qual o serviço está localizado pode mudar ou porque ele deve ser ocultado para controlar o acesso ao serviço
  • Para suportar o acesso ao serviço sobre HTTP ou HTTPS
  • Quando desejar fornecer criação de log de todos os pedidos ao provedor de serviços ou quando desejar fornecer recursos de manipulação de erros adicionais sem fazer alterações no provedor de serviços

Solução

A solução é implementar um fluxo de mensagens que forneça um proxy para o serviço da Web do provedor e também possa incluir recursos de criação de log e manipulação de erros.
