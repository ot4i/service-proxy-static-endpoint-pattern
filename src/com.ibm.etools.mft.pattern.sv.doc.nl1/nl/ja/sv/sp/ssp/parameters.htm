「サービス・プロキシー: 静的エンドポイント」パターンのパラメーター

サービス情報
パターン・パラメーター M C デフォルト 説明
サービス・プロバイダー用の WSDL はい いいえ  

このパラメーターは、プロキシーを作成するサービスのインターフェースを定義するデプロイ可能な WSDL ファイルを定義します。

デプロイ可能な WSDL ファイルをワークスペース・メッセージ・セットから選択する必要があります。
サービス・プロバイダーの URL いいえ はい  

このパラメーターは、プロキシーを作成するサービスが存在するエンドポイントを定義します。

指定されていない場合、エンドポイントは WSDL ファイルのバインディング情報から決定されます。
プロキシー URL いいえ はい  

このパラメーターは、デプロイされるプロキシーのエンドポイントを定義するために使用される URL パスを定義します。

完全なサービス・アドレスには、ホスト名およびポートも含まれます。これは、プロキシーがデプロイされるときに決定されます。

パスが指定されていない場合、サービス WSDL のパスが使用されます。
SOAP 要求の妥当性検査 いいえ はい なし

着信 SOAP メッセージは、このパラメーターで定義されたとおりに妥当性検査されます。

有効な値は以下のとおりです。

  • なし
  • 内容と値
SOAP 応答の妥当性検査 いいえ はい なし

SOAP 応答は、このパラメーターで定義されたとおりに妥当性検査されます。

有効な値は以下のとおりです。

  • なし
  • 内容と値
ロギング
パターン・パラメーター M C デフォルト 説明
必要なロギング いいえ いいえ クリア済み

このパターン・パラメーターは、ロギング用のコードをパターン・インスタンスに含めるかどうかを決定します。

「必要なロギング」が非選択の場合、ログ・サブフローは含まれません。

「必要なロギング」が選択されている場合、ログ・サブフローがメッセージ・フローに含まれ、「ログ・キュー・マネージャー」と「ログ・キュー」の値を設定する必要があります。
ログ・キュー・マネージャー いいえ はい

このパターン・パラメーターは、ロギング用のキュー・マネージャーを定義します。 ロギングにブローカー・キュー・マネージャーが使用される場合、これをブランクのままにすることができます。

このパラメーターは、「必要なロギング」が選択されている場合にのみ必要です。

ログ・キュー いいえ はい LOG

このパターン・パラメーターは、ロギング用のキューを定義します。

このパラメーターは、「必要なロギング」が選択されている場合にのみ必要です。

エラー処理
パターン・パラメーター M C デフォルト 説明
必要なエラー・メッセージ いいえ いいえ 選択済み

このパターン・パラメーターは、エラー・メッセージを作成するメッセージ・フロー・エレメントがパターン適用によって作成されるかどうかを定義します。

「必要なエラー・メッセージ」が選択されている場合、「エラー・キュー・マネージャー」および「エラー・キュー」の値を設定する必要があります。

エラー・キュー・マネージャー いいえ はい  

このパターン・パラメーターは、エラー・メッセージ用のキュー・マネージャーを定義します。 ロギングにブローカー・キュー・マネージャーが使用される場合、これをブランクのままにすることができます。

エラー・キュー いいえ はい ERROR

このパターン・パラメーターは、エラー・メッセージ用のキューを定義します。

このパラメーターは、必要なエラー・メッセージ が選択されている場合のみ必要です。

その他
パターン・パラメーター M C デフォルト 説明
ブローカー・スキーマ いいえ いいえ mqsi

ブローカー・スキーマが定義されている場合、メッセージ・フローおよび ESQL ファイルがそのスキーマ内に作成されます。 スキーマが定義されていない場合、デフォルトのブローカー・スキーマが使用されます。
フロー接頭部 いいえ いいえ  

設定されている場合、この接頭部はすべてのメッセージ・フロー名の先頭に追加されます。

メッセージ・フロー名で有効な文字のみを使用できます。
フロー接尾部 いいえ いいえ  

設定されている場合、この接尾部はすべてのメッセージ・フロー名の末尾に追加されます。

メッセージ・フロー名で有効な文字のみを使用できます。
キュー接頭部 いいえ いいえ  

設定されている場合、この接頭部はすべてのキュー名の先頭に追加されます。

デフォルトのキュー名が使用されている場合、各パターン・インスタンスのキュー名を識別するために接頭部、接尾部、またはその両方を設定する必要があります。

キュー接尾部 いいえ いいえ  

設定されている場合、この接尾部はすべてのキュー名の末尾に追加されます。

デフォルトのキュー名が使用されている場合、各パターン・インスタンスのキュー名を識別するために接頭部、接尾部、またはその両方を設定する必要があります。

簡略説明 いいえ いいえ  

この説明は、生成されるすべてのメッセージ・フローの「簡略説明」プロパティーに追加されます。

詳細説明 いいえ いいえ  

この説明は、生成されるすべてのメッセージ・フローの「詳細説明」プロパティーに追加されます。

すべてのパターン・パラメーターとそのデフォルト値をリストします。

M という見出しの列は、プロパティーが必須 かどうかを示します。C という見出しの列は、プロパティーが構成可能かどうか、つまり、メッセージ・フローをブローカー・アーカイブ (BAR) ファイルに追加するときに値を変更できるかどうかを示します。

デフォルトが示されていない場合、デフォルト値は空ストリングです。
