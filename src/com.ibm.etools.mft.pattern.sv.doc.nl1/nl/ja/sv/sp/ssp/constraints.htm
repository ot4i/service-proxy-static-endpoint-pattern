「サービス・プロキシー: 静的エンドポイント」パターンの使用に関する制約

このパターンを使用する場合に適用される制約は多数あります。

以下のすべての制約を満たすことができる場合にのみ、「サービス・プロキシー: 静的エンドポイント」パターンを使用できます。

  • HTTP または HTTPS を介した SOAP メッセージとして要求を受け取る必要があります。
  • 要求元アプリケーションが同期要求応答対話スタイルをサポートしている必要があります。
  • ターゲット・サービスが SOAP over HTTP または HTTPS バインディングを提供している必要があります。
  • ターゲット・サービス・インターフェースを定義する WSDL には複数のポート・タイプとバインディングを含めることができますが、生成されたパターン・インスタンスの構成には、最初のポート・タイプまたはバインディングのみが使用されます。 メッセージ・フロー・エディターで代わりのものを選択できます。
  • ロギングが必要な場合、このパターンで使用されるフォーマットが、実際の環境の要件を満たしていると想定されます。
